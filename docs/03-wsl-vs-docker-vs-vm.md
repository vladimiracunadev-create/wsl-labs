# 03 - WSL vs Docker vs Máquina Virtual

## WSL

Ideal para tener un entorno Linux integrado con Windows.

Usos comunes:

- Desarrollo local.
- Scripts Linux.
- Pruebas de servicios.
- Entornos de aprendizaje.
- Herramientas DevOps.

## Docker

Ideal para empaquetar aplicaciones con sus dependencias y ejecutarlas como contenedores.

Usos comunes:

- Microservicios.
- Ambientes reproducibles.
- Despliegues.
- CI/CD.
- Laboratorios aislados.

## Máquina virtual

Ideal cuando se necesita un sistema operativo completo, aislado y más parecido a un servidor tradicional.

Usos comunes:

- Simular servidores completos.
- Pruebas de redes.
- Sistemas con kernel propio.
- Ambientes más aislados.

## Comparación rápida

| Tecnología | Mejor uso |
|---|---|
| WSL | Linux integrado a Windows |
| Docker | Aplicaciones en contenedores |
| VM | Sistema operativo completo aislado |
| Unikernel | Sistema mínimo especializado |
